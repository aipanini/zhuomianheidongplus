# 桌面黑洞Plus

在桌面上呈现一个可跟随鼠标的真实黑洞：捕获真实桌面并经 GLSL 点质量引力透镜着色器扭曲，
中心为事件视界（纯黑），周围是旋转、带多普勒红移/蓝移的吸积盘。窗口鼠标穿透且被排除出屏幕捕获，
因此你能看到「桌面被黑洞弯曲」的同时，仍可正常操作真实桌面——如同电脑里真有一个黑洞。

## 效果预览

- 真实引力透镜扭曲（爱因斯坦环 + 次级像）
- 5种视觉风格：金色 / 炽焰 / 经典橙红 / 幽冥青 / 紫电
- 旋转吸积盘 + 多普勒红移/蓝移 + 软发光晕
- 弹簧阻尼物理跟随鼠标
- 桌面图标吞噬（靠近时被吸引进黑洞，可恢复）

## 环境要求

- Windows 10 2004+ / Windows 11（捕获排除 `WDA_EXCLUDEFROMCAPTURE` 需要）
- Python 3.8+
- 依赖：`pip install -r requirements.txt`

## 运行

```bash
python main.py
```

启动后桌面中央出现黑洞，缓慢跟随鼠标移动。

## 控制

### 快捷键（推荐）

基于 `GetAsyncKeyState` 轮询实现，不依赖消息队列，在鼠标穿透模式下也能稳定响应：

| 快捷键 | 作用 |
| --- | --- |
| **`F9`** | 显示 / 隐藏黑洞（单键，最顺手；不退出程序） |
| `F2` | 打开参数控制面板 |
| `Ctrl+Alt+Space` | 显示 / 隐藏黑洞（三键版） |
| `Ctrl+Alt+↑` | 增大黑洞 |
| `Ctrl+Alt+↓` | 缩小黑洞 |
| `Ctrl+Alt+Q` | 退出程序 |

### 系统托盘

启动后任务栏通知区会出现黑洞图标，**右键**点击即可弹出菜单：

| 菜单项 | 作用 |
| --- | --- |
| 显示 / 隐藏黑洞 | 平滑淡入淡出 |
| 增大 / 缩小 | 调整黑洞大小 |
| 重置大小 | 恢复默认尺寸 |
| 退出 | 关闭程序 |

> **找不到托盘图标？** Windows 11 默认将托盘图标收起到溢出区，点击任务栏右下角的 **`^`** 箭头展开即可看到黑洞图标。
> 若热键已可用，无需依赖托盘——`F9` 切换显隐、`Ctrl+Alt+Q` 退出即可。

> 鼠标穿透模式下窗口无焦点，故用全局快捷键。若在 `config.py` 中将 `CLICK_THROUGH` 设为 `False`，
> 窗口可获得焦点，此时也可用键盘 `ESC / 空格 / ↑ / ↓` 控制。

## 参数调节

所有可调参数集中在 [config.py](config.py)，也可以按 `F2` 打开可视化控制面板实时调节：

- `EVENT_HORIZON` / `EINSTEIN_RADIUS`：黑洞大小与透镜扭曲强度
- `WARP_BOOST`：广域扭曲增强，让远处文字也明显弯曲（0=纯物理透镜，1=广域可见扭曲）
- `DISK_BRIGHTNESS`：吸积盘亮度
- `MOUSE_PULL`：鼠标吸入强度
- `STYLE`：视觉风格（gold/fire/classic/ghost/purple）
- `FOLLOW_STIFFNESS` / `FOLLOW_DAMPING`：跟随速度与惯性（弹簧 + 阻尼）
- `SWALLOW_*`：吞噬功能参数
- `CLICK_THROUGH` / `TOPMOST` / `EXCLUDE_FROM_CAPTURE`：窗口行为

控制面板提供4种预设：微型、标准、巨型、狂暴。

## 工作原理

1. 创建全屏、置顶、鼠标穿透、捕获排除的无边框 OpenGL 窗口；
2. 每帧用 `mss` 捕获真实桌面（本窗口被排除，故无反馈拖影）并上传为 GPU 纹理；
3. 片段着色器按点质量引力透镜方程 `β = θ(1 - θ_E²/|θ|²)` 对背景采样坐标进行扭曲，
   自然形成爱因斯坦环与次级像；
4. 叠加事件视界（纯黑）与吸积盘（径向渐变 + 旋转螺旋 + 多普勒红蓝移 + 软发光晕）；
5. 黑洞位置以弹簧-阻尼平滑跟随鼠标，模拟物理惯性；
6. 吞噬模块通过 Windows API 获取桌面图标真实位置，靠近时螺旋吸入并移动文件。

## 常见问题

- **屏幕全黑**：少数显卡在「鼠标穿透 + OpenGL」下渲染异常。将 `config.py` 的 `CLICK_THROUGH` 改为 `False`，
  改用普通置顶窗口（可获焦点、用键盘控制）。
- **黑洞周围有拖影/残像**：说明系统不支持捕获排除（Win10 2004 以下）。升级系统，或将 `EXCLUDE_FROM_CAPTURE` 设为 `False`
  （会引入轻微反馈，仅作降级方案）。
- **热键无效**：可能被其他程序占用。修改 `config.py` 中的 `HOTKEY_*` 即可。
- **性能不足**：调低 `TARGET_FPS`，或缩小 `EVENT_HORIZON`/`EINSTEIN_RADIUS`。
- **吞噬功能不工作**：确保以普通用户权限运行（非管理员），部分系统桌面图标布局可能需要手动刷新。
